## 差错产生与类型

### 差错产生的原因

> **传输差错**
> 
> 数字信号在物理介质中传输时，由于干扰导致接收端收到的比特与发送端不同。

#### 差错产生机制

```
  发送方                信道                  接收方
11010110  ──原始──>  [噪声/衰减]  ──受扰──>  11110110
                                              ↑
                                           第3位翻转
```

**主要干扰源**：
- 热噪声：电子元件热运动产生
- 脉冲噪声：瞬时强干扰信号
- 串扰：相邻信道信号泄露
- 信号衰减：长距离传输功率损失

### 差错类型分析

#### 1. 按错误模式分类

**单比特差错**：
- 只有一个比特发生错误
- 在高质量传输介质中较常见
- 相对容易检测和纠正

**突发差错**：
- 连续多个比特发生错误
- 在无线和长距离传输中常见
- 检测和纠正难度较大

#### 2. 差错分布特性

| 差错类型 | 特征 | 产生原因 | 检测难度 |
|---------|------|---------|---------|
| 随机差错 | 分布均匀 | 热噪声 | 中等 |
| 突发差错 | 集中分布 | 脉冲干扰 | 较高 |
| 周期差错 | 规律出现 | 时钟漂移 | 较低 |

## 奇偶校验技术

### 单比特奇偶校验

> **单比特奇偶校验**
> 
> 在数据比特后添加一个校验比特，使整个数据（含校验比特）中 1 的个数为奇数（奇校验）或偶数（偶校验）。

#### 工作原理

偶校验示例（让总的 1 的个数为偶数）：

```
 数据位        校验位     发送
1101001  (4个1) → 0  →  11010010   (1的个数仍为偶)
1101011  (5个1) → 1  →  11010111   (补成偶)
└─ d0..d6 ─┘     P
```

校验位计算（设数据位为 $d_0 \dots d_{n-1}$）：

$$P_{\text{偶}} = d_0 \oplus d_1 \oplus \cdots \oplus d_{n-1}, \qquad P_{\text{奇}} = P_{\text{偶}} \oplus 1$$

#### 检错能力

- 能检测所有奇数个比特错误。
- 偶数个比特错误会相互抵消，无法检出。
- 不能纠正错误；对突发错误（常成对翻转）效果差。

注：单比特奇偶校验只能检错、不能纠错——它只知道"出错了"，不知道"错在哪一位"。

### 二维奇偶校验

> **二维奇偶校验**
> 
> 将数据排成二维矩阵，对每行、每列各加一个偶校验位。单比特错误会同时引起一行、一列的校验失败，由行列交点定位并纠正。

#### 实现方法

$4\times4$ 数据矩阵，每行每列取偶校验（行/列校验位 = 该行/列数据位异或）：

```
         c1 c2 c3 c4 | 行校验
       +-------------+------
   r1  | 1  0  1  1  |  1
   r2  | 1  1  0  1  |  1
   r3  | 0  1  1  0  |  0
   r4  | 1  0  0  1  |  0
       +-------------+------
列校验 | 1  0  0  1  |  0    ← 右下角 = 所有校验位异或
```

#### 错误检测与定位

设接收时第 2 行第 3 列发生翻转（1→0）：

```
         c1 c2 c3 c4 | 行校验  重算  结果
       +-------------+------------------
   r1  | 1  0  1  1  |  1       1    OK
   r2  | 1  1 [0] 1  |  1       0    行校验失配 ←
   r3  | 0  1  1  0  |  0       0    OK
   r4  | 1  0  0  1  |  0       0    OK
       +-------------+------------------
列校验 | 1  0  0  1  |
重算   | 1  0  1  1  |
结果   |OK OK 失配 OK|              第3列失配 ↑
```

行 2 与列 3 同时失配 → 错误位于交点 (r2, c3)，将该位取反即可纠正。

#### 纠检错能力

- 纠正任意单比特错误（行列交点唯一确定）。
- 检测任意双比特错误（双错不会同时被一行一列唯一定位，故只检不纠）。
- 检测大部分多比特错误。

易混：单比特奇偶校验只检错；二维奇偶校验能纠单错、检双错——升一维带来了"定位"能力。

## 检验和方法

### Internet检验和

> **Internet 检验和**
> 
> 将数据看作 16 比特字的序列，做"带回卷进位"的二进制求和，再取反码作为检验和。

#### 计算过程

发送端：

1. 将数据分割为 16 比特字。
2. 逐个求和，最高位的进位回卷加到最低位（end-around carry）。
3. 对回卷和取反码，作为检验和一并发送。

示例（三个 16 比特字相加）：

```
  0100010101000100
+ 0101010101010101
+ 1000111100001111
  ----------------
和(含进位回卷后) = 0010100110101001
检验和(取反码)   = 1101011001010110
```

注：本例三字相加产生进位，须回卷后再取反；直接丢弃进位会得到错误结果。

#### 检错原理

接收端把所有 16 比特字连同检验和一起求和（同样回卷）：结果全 1 则无错，否则有错。

```
回卷和 + 检验和 = 0010100110101001 + 1101011001010110 = 1111111111111111
```

检错特性：
- 能检测所有单比特错误，多比特错误大部分可检出。
- 计算简单，软件即可实现；但检错能力弱于 CRC。
- 不能纠正错误。

## 循环冗余检测CRC

### CRC基本原理

> **循环冗余检测（CRC）**
> 
> 把数据看作二进制多项式的系数，用预定的生成多项式做模 2 除法，余数即冗余校验码。

#### 数学基础

多项式表示：数据位串对应多项式系数，例如 $110001 \to x^5 + x^4 + 1$。

生成多项式 $G(x)$：

- $r+1$ 位的二进制模式（最高位、最低位通常为 1），阶数为 $r$。
- 决定 CRC 的检错能力，余数固定为 $r$ 位。
- 常用标准：CRC-8、CRC-16、CRC-32。

#### CRC 计算过程

设数据 $D$ 为 $d$ 位，生成多项式 $G$ 为 $r+1$ 位：

```
   D (d位)
     │ 左移 r 位 → D·2^r
     ▼
  (D·2^r) ÷ G   ── 模2除法(异或) ──>  余数 R (r位)
     │                                   │
     └──────────── 拼接 ────────────────┘
                     ▼
              传输帧 = D 后接 R  (d+r 位)
```

模 2 除法用按位异或代替减法、无借位；接收端用同一 $G$ 除收到的帧，余数为 0 则无错。

### CRC编码实现

#### 发送端编码

**步骤**：

1. 数据准备：原始数据 $D$（$d$ 位）。
2. 左移：$D$ 左移 $r$ 位得 $D \cdot 2^r$（末尾补 $r$ 个 0）。
3. 模 2 除法：$(D \cdot 2^r) \div G$，得余数 $R$（$r$ 位，不足补前导 0）。
4. 组合传输：发送 $D$ 后接 $R$（$d+r$ 位）。

计算示例：

```
数据 D = 1101 (4位)，生成多项式 G = 1011 (r=3)

D 左移 3 位 → 1101000，对其做模2除法：

        1111         ← 商(可不写)
       _______
1011 ) 1101000
       1011          1101 XOR 1011 = 0110
       ----
        1100         下移得 1100，XOR 1011 = 0111
        1011
        ----
         1110        下移得 1110，XOR 1011 = 0101
         1011
         ----
          1010       下移得 1010，XOR 1011 = 0001
          1011
          ----
           001       余数 R = 001

传输帧 = 1101 + 001 = 1101001
```

注：余数必须补足为 $r=3$ 位，故写作 001 而非 1。

#### 接收端检验

用相同生成多项式 $G$ 除接收帧：余数为 0 则无错，非 0 则有错。本例 $1101001 \div 1011$ 余数为 000，校验通过。

### 常用 CRC 标准

| 标准 | 阶数 $r$ | 生成多项式 | 应用场景 |
|------|---------|-----------|----------|
| CRC-8 | 8 | $x^8+x^2+x+1$ | 简单应用 |
| CRC-16 | 16 | $x^{16}+x^{15}+x^2+1$ | 磁盘、串口 |
| CRC-32 | 32 | IEEE 802 标准多项式 | 以太网、ZIP |

### CRC 检错能力

只要 $G(x)$ 选取得当，CRC 可保证检出：

- 所有单比特错误（$G(x)$ 至少含两项时成立）。
- 所有奇数个比特错误（$G(x)$ 含因子 $x+1$ 时成立）。
- 长度 $\le r$ 的所有突发错误。

对长度 $>r$ 的突发错误及随机错误，漏检概率约为 $2^{-r}$，即检出概率约 $1-2^{-r}$。

易混：CRC 检错能力很强，但只能检错、不能纠错——余数非 0 只说明帧出错，无法定位错在哪一位。

### CRC计算典型例题

#### 例题1：CRC 编码计算

> **题目**：数据 101001，生成多项式 $G(x) = x^4 + x + 1$，求 CRC 码。

$G(x) = x^4 + x + 1 \to 10011$（5 位，$r = 4$）。数据左移 4 位：$101001 \to 1010010000$。

模 2 除法：

```
        101110         ← 商
       ___________
10011 ) 1010010000
        10011          10100 XOR 10011 = 00111
        -----
        001110         下移，首位0 → 商0
         11101         下移得 11101，XOR 10011 = 01110
         10011
         -----
          11010        下移得 11010，XOR 10011 = 01001
          10011
          -----
           10010       下移得 10010，XOR 10011 = 00001
           10011
           -----
             0010      下移补0，余数 R = 0010
```

CRC 码 = 数据 + 余数 = $101001 + 0010 = 1010010010$。

注：余数为 0010（4 位），不可省成 10。验证：$1010010010 \div 10011$ 余 0，正确。

#### 例题2：CRC 检错验证

> **题目**：收到 1010011001，生成多项式 10011，判断是否有错。

用 $G = 10011$ 除接收帧：

```
        101110         ← 商
       ___________
10011 ) 1010011001
        10011          10100 XOR 10011 = 00111
        -----
        001111         首位0 → 商0；下移得 11111
         11111
         10011         11111 XOR 10011 = 01100
         -----
          11000        下移得 11000，XOR 10011 = 01011
          10011
          -----
           10110       下移得 10110，XOR 10011 = 00101
           10011
           -----
             1011      下移补0，余数 = 1011
```

余数 $1011 \neq 0$，故传输有错。（对照例题1，正确码字应为 1010010010。）

#### 例题3：考研真题风格计算

> **题目**：数据 1101011，生成多项式 $G(x) = x^3 + x + 1$，求：
> 1. 发送的 CRC 码字；
> 2. 若收到码字 1101011100，判断是否正确。

**第 1 问**：计算 CRC 码字

$G(x) = x^3 + x + 1 \to 1011$（$r = 3$）。数据左移 3 位：$1101011 \to 1101011000$。

```
        1111010        ← 商
       ___________
1011 ) 1101011000
       1011            1101 XOR 1011 = 0110
       ----
        1100           下移得 1100，XOR 1011 = 0111
        1011
        ----
         1111          下移得 1111，XOR 1011 = 0100
         1011
         ----
          1001         下移得 1001，XOR 1011 = 0010
          1011
          ----
            1000       下移得 1000，XOR 1011 = 0011
            1011
            ----
             110       余数 R = 110
```

余数 $R = 110$，CRC 码字 = $1101011 + 110 = 1101011110$。

**第 2 问**：验证收到的 1101011100

用 $G = 1011$ 除 1101011100，余数为 $010 \neq 0$，故传输有错（正确码字为 1101011110，可见第 8 位被翻转）。

## 前向纠错FEC技术

### FEC基本概念

> **前向纠错（FEC）**
> 
> 发送端加入足够的冗余信息，使接收端不仅能检错，还能直接纠错，无需重传。

#### 基本思想

```
 信息源       编码器          噪声信道        译码器       信息宿
 k比特  ──>  加冗余  ──n比特码字(n>k)──>  [可能出错]  ──>  检错纠错  ──>  k比特
```

用 $n-k$ 位冗余换取纠错能力，省去重传往返时延，适合不便重传的场景（如卫星、广播、存储）。

### 海明码原理

> **海明码**
> 
> 能纠正单比特错误的线性分组码。校验位放在 $2^i$ 位置，让校验失败的组合恰好编码出错位的位置。

#### 海明码构造

校验位放在位置 $2^i$（即 1, 2, 4, 8, …），其余为数据位。(7,4) 海明码布局：

```
位置:  1   2   3   4   5   6   7
内容:  P1  P2  D1  P3  D2  D3  D4
       └校验位┘  └─── 数据位 ───┘
```

每个校验位覆盖"位置二进制中对应位为 1"的所有位置：

```
位置  二进制   参与的校验位
 1     001       P1
 2     010          P2
 3     011       P1 P2
 4     100             P3
 5     101       P1    P3
 6     110          P2 P3
 7     111       P1 P2 P3

P1 覆盖位置 1,3,5,7    (含 P1,D1,D2,D4)
P2 覆盖位置 2,3,6,7    (含 P2,D1,D3,D4)
P3 覆盖位置 4,5,6,7    (含 P3,D2,D3,D4)
```

#### 编码过程

传输数据 $D_1D_2D_3D_4 = 1011$，按偶校验令各校验位使其覆盖范围内 1 的个数为偶：

```
位置:   1   2   3   4   5   6   7
内容:   P1  P2  1   P3  0   1   1
        ↑   ↑       ↑
       D1=1 D2=0 D3=1 D4=1
```

$$P_1 = D_1 \oplus D_2 \oplus D_4 = 1\oplus0\oplus1 = 0$$
$$P_2 = D_1 \oplus D_3 \oplus D_4 = 1\oplus1\oplus1 = 1$$
$$P_3 = D_2 \oplus D_3 \oplus D_4 = 0\oplus1\oplus1 = 0$$

最终码字（位置 1–7）：$0\,1\,1\,0\,0\,1\,1 = $ **0110011**。

#### 纠错过程

设接收时第 3 位翻转（1→0），收到 **0100011**。计算三个校正子（各覆盖范围异或）：

$$S_1 = P_1 \oplus D_1 \oplus D_2 \oplus D_4 = 0\oplus0\oplus0\oplus1 = 1$$
$$S_2 = P_2 \oplus D_1 \oplus D_3 \oplus D_4 = 1\oplus0\oplus1\oplus1 = 1$$
$$S_3 = P_3 \oplus D_2 \oplus D_3 \oplus D_4 = 0\oplus0\oplus1\oplus1 = 0$$

校正子 $S_3S_2S_1 = 011_2 = 3$，正好指出第 3 位出错；将其取反即恢复原码字。

注：校正子的二进制值直接等于出错位置，这正是把校验位放在 $2^i$ 位置的目的。若校正子全 0 则无错。

#### 海明距离与纠检错能力

> **海明距离**
> 
> 两个等长码字对应位不同的个数，记 $d(x,y)$。一个码的**最小海明距离** $d_{\min}$ 是任意两个合法码字之间海明距离的最小值。

错误把合法码字"推"开 $e$ 位。能否发现/还原取决于它是否越过相邻码字的"中线"：

$$d_{\min} \ge e + 1 \;\Rightarrow\; \text{可检测 } e \text{ 位错}$$
$$d_{\min} \ge 2e + 1 \;\Rightarrow\; \text{可纠正 } e \text{ 位错}$$

直观理解：

- 检 $e$ 位错，只需保证 $e$ 位翻转后不会"碰巧"变成另一个合法码字，故合法码字间至少要隔 $e+1$。
- 纠 $e$ 位错，要让出错码字离原码字仍比离其他任何码字都近，故至少要隔 $2e+1$。

| $d_{\min}$ | 可检错位数 $e=d_{\min}-1$ | 可纠错位数 $e=\lfloor(d_{\min}-1)/2\rfloor$ |
|:---:|:---:|:---:|
| 3 | 2 | 1 |
| 4 | 3 | 1 |
| 5 | 4 | 2 |

(7,4) 海明码 $d_{\min}=3$，故可纠 1 位、检 2 位。

易混：同一 $d_{\min}$ 下，检错能力总比纠错能力强——发现错误比定位并还原错误容易。$d_{\min}=3$ 时若全用于检错可检 2 位，若用于纠错则只能纠 1 位，二者不可兼得到上限。

## 差错控制技术对比

### 技术特性对比

| 技术 | 检错能力 | 纠错能力 | 冗余开销 | 典型应用 |
|------|---------|---------|---------|----------|
| 单比特奇偶 | 仅奇数位错 | 无 | 1 比特 | 简单串口 |
| 二维奇偶 | 检双错 | 纠单错 | 行+列 | 教学/简单存储 |
| 检验和 | 中等 | 无 | 16 位 | Internet 协议 |
| CRC | 强 | 无 | 8–32 位 | 数据链路层、以太网 |
| 海明码 | 检双错 | 纠单错 | $\approx\log_2 n$ 位 | 内存 ECC |

冗余率（冗余位数 / 总位数）：

- 奇偶校验：$\dfrac{1}{n}$，数据越长冗余越低。
- CRC-r：$\dfrac{r}{d}$（$r$ 固定），对长帧效率高。
- (n,k) 海明码：约 $\dfrac{\log_2 n}{n}$，短码字效率偏低。

### 选择策略

按链路误码率（BER）权衡效率与可靠性：

- 高质量链路（BER < $10^{-6}$）：CRC 检错 + 出错重传即可，开销最低。
- 中等质量链路（BER ≈ $10^{-4}$）：CRC + ARQ 重传，或加简单纠错码。
- 低质量链路（BER > $10^{-3}$）：用强纠错码（如卷积码）配合交织对抗突发错误，可靠性优先。

链路层多用 CRC（只检错、配合重传）；存储器、卫星等不便重传的场景才用 FEC 纠错码。
