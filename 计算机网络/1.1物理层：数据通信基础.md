## 数据通信基础概念

### 通信系统模型

任何一次数据通信都可抽象成下面这条链路：信源产生数据，发送器把数据变成适合信道传输的信号，信号经过信道（途中混入噪声）到达接收器，接收器还原出数据交给信宿。

```
  信源 ──数据──> 发送器 ──信号──> [   信   道   ] ──信号──> 接收器 ──数据──> 信宿
 (Source)      (Transmitter)        │              (Receiver)        (Sink)
                                  噪声源
                              (热噪声/串扰/干扰)
```

- **信源 / 信宿**：数据的产生者与最终接收者（如两台主机的应用）。
- **发送器 / 接收器**：完成"数据↔信号"的变换，如调制解调器（Modem）、网卡的物理层芯片。
- **信道（Channel）**：信号传播的通路，可以是一段物理介质，也可以是介质上划分出的一个逻辑子通路（如一个频段、一个时隙）。
- **噪声源（Noise）**：信道中不可避免的干扰，是限制传输速率的根本原因（见[信道容量](#信道容量与传输极限)）。

注：物理介质（一根光纤）和信道（介质上的一条通路）不是一回事——一根介质上可以用复用技术划出多条信道。

### 数据、信号、码元、比特

这几个术语常被混用，先界定清楚：

| 术语 | 含义 |
|------|------|
| 数据（Data） | 承载信息的实体，分**模拟数据**（连续，如声音）与**数字数据**（离散，如 0/1 串） |
| 信号（Signal） | 数据的电磁/光学表示，分**模拟信号**（连续波形）与**数字信号**（离散电平） |
| 码元（Symbol） | 信号上携带信息的**基本波形单元**，一个码元可表示 1 个或多个比特 |
| 比特（Bit） | 信息量的基本单位，取值 0 或 1 |

一个码元能表示几个比特，取决于它有多少种可区分的状态 $M$（电平数/相位数/相位幅度组合数）：一个码元携带 $\log_2 M$ 个比特。

```
 M=2 (二电平)         M=4 (四电平)
  1 ─┐  ┌─            11 ───
     │  │             10 ──
  0  └──┘             01 ─
 1码元=1bit           00 
                     1码元=2bit
```

### 波特率 vs 比特率

这是物理层最容易混的一对概念。

> **码元速率（波特率，Baud Rate）** $R_B$：每秒传输的**码元个数**，单位 Baud（波特）。又称符号速率、调制速率。
>
> **信息速率（比特率，Bit Rate）** $R_b$：每秒传输的**比特数**，单位 bps（bit/s）。又称数据率。

两者的关系由"每码元几个比特"决定：

$$R_b = R_B \log_2 M \quad(\text{bps})$$

其中 $M$ 为一个码元的可取状态数（进制数）。

- 当 $M = 2$（二进制，如 NRZ）：$R_b = R_B$，波特率等于比特率，常被混称——但这只是 $M=2$ 的巧合。
- 当 $M > 2$（如 16-QAM，$M=16$）：一个码元载 4 bit，比特率是波特率的 4 倍。

**举例**：某调制方案波特率 $R_B = 2000$ Baud，采用 16-QAM（$M=16$）：

$$R_b = 2000 \times \log_2 16 = 2000 \times 4 = 8000 \text{ bps}$$

注：提高 $M$ 能在不增加波特率（即不占用更多带宽）的前提下提速，但 $M$ 越大、电平/相位间隔越密，越容易被噪声"认错"，所以 $M$ 不能无限增大——上限由香农定理给出。

易混：曼彻斯特编码每个比特中间有一次跳变，其信号变化率（调制速率）是数据率的 **2 倍**，即波特率 = 2×比特率，这是"波特率 > 比特率"的典型反例。

### 通信方式

按数据传输方向分：

| 方式 | 方向 | 比喻 | 例子 |
|------|------|------|------|
| 单工（Simplex） | 单向，固定收/发 | 广播 | 无线电广播、键盘到主机 |
| 半双工（Half-duplex） | 双向，但不能同时 | 对讲机 | 早期对讲、共享介质以太网（CSMA/CD） |
| 全双工（Full-duplex） | 双向，可同时 | 电话 | 交换式以太网、现代有线链路 |

按数据位的组织方式分：

- **串行传输**：比特在一根线上一位接一位地发。线少、成本低、抗串扰，适合长距离——网络通信几乎全用串行。
- **并行传输**：多个比特用多条线同时发。速度快但线多、长距离上各线时延不齐（位偏移）难同步，多用于设备内部短距离总线。

### 基带传输 vs 宽带传输

> **基带（Baseband）传输**：直接把数字信号（方波，占据从 0 到某上限的低频段）送上信道，不经载波调制。
>
> **宽带（Broadband）传输**：把信号调制到某个高频载波上再传，可在同一介质上用频分复用承载多路。

- 基带：信号简单、设备便宜，但方波含丰富高频分量、衰减快，传输距离受限。**以太网在双绞线/光纤上就是基带传输**。
- 宽带：能频分复用、传得远，但需调制解调。**有线电视网（HFC）、DSL** 是宽带传输的代表。

注：这里的"基带/宽带"是物理层的信号传输方式，与日常说"宽带上网"（指接入速率高）含义不同，别混。

## 传输介质与信道

[1.0 物理媒体](1.0%20计算机网络和互联网.md) 已介绍各介质"是什么、用在哪"，这里侧重**物理特性与选型对比**，不重复接入网场景。介质分两大类：

```
传输介质
├─ 导引型(有线)：信号被约束在固体介质中
│   ├─ 双绞线 (UTP / STP)
│   ├─ 同轴电缆
│   └─ 光纤 (单模 / 多模)
└─ 非导引型(无线)：信号在空间自由传播
    ├─ 无线电波
    ├─ 微波 (地面中继 / 卫星)
    └─ 红外 / 可见光
```

### 双绞线（Twisted Pair）

两根绝缘铜线按一定密度互相绞合——绞合能让相邻线对的电磁干扰相互抵消，绞得越密、抗干扰越好。

- **UTP（非屏蔽双绞线）**：无金属屏蔽层，便宜、易布线，是局域网主流。
- **STP（屏蔽双绞线）**：加了金属屏蔽层，抗干扰更强、更贵，用于强干扰环境。

UTP 按性能分类（"类别"对应可支持的带宽/速率）：

| 类别 | 带宽 | 典型应用 |
|------|------|---------|
| Cat3 | 16 MHz | 早期 10BASE-T、电话 |
| Cat5 | 100 MHz | 100BASE-TX 快速以太网 |
| Cat5e | 100 MHz | 1000BASE-T 千兆以太网 |
| Cat6 | 250 MHz | 千兆/万兆（短距） |
| Cat6a | 500 MHz | 10GBASE-T 万兆 |

### 同轴电缆（Coaxial Cable）

内导体、绝缘层、网状外导体、外护套同轴排列。外导体兼作屏蔽，抗干扰和带宽都优于双绞线。曾用于早期总线型以太网（10BASE2/5）和有线电视，如今局域网中已基本被双绞线/光纤取代，主要留在有线电视接入（HFC）。

```
 ┌─────────────────────────┐  外护套
 │ ┌─────────────────────┐ │  网状外导体(屏蔽)
 │ │ ┌─────────────────┐ │ │  绝缘层
 │ │ │ ───────────────  │ │ │  内导体(铜芯)
 │ │ └─────────────────┘ │ │
 │ └─────────────────────┘ │
 └─────────────────────────┘
```

### 光纤（Optical Fiber）

利用光的全反射在玻璃纤芯中传光。带宽极高、衰减极小、抗电磁干扰、不易窃听，是长距离骨干和高速链路的首选。按纤芯直径和传播模式分两种：

| 类型 | 纤芯 | 光源 | 传播 | 距离 | 成本 |
|------|------|------|------|------|------|
| 单模（SMF） | 细（约 9 μm） | 激光 | 单一路径，几乎无模间色散 | 长（数十~上百 km） | 高 |
| 多模（MMF） | 粗（50/62.5 μm） | LED | 多条光路，存在模间色散 | 短（数百 m） | 低 |

```
 多模：光线多路径反射，到达有先后(模间色散，限制距离)
   ───\    /\    /\    /───
       \  /  \  /  \  /
        \/    \/    \/

 单模：纤芯极细，光近乎直线单路传播
   ─────────────────────────
```

注：单模"贵"在激光器和精密对准，光纤本身比多模还便宜；远距离/高速选单模，楼内短距选多模。

### 无线介质

无引导路径，信号在空间扩散，特性随频段变化：

- **无线电波（< 1 GHz）**：全向辐射、能绕射、穿墙能力较好，用于广播、Wi-Fi（2.4 GHz）、蜂窝。
- **微波（1~40 GHz）**：方向性强，需视距（LOS）传播。
  - **地面微波**：用塔间天线对准中继，跨越无法铺缆的地形。
  - **卫星微波**：以卫星作中继，覆盖广。按轨道高度分：

| 卫星轨道 | 高度 | 单跳时延（约） | 特点 |
|---------|------|--------------|------|
| GEO 同步 | ~35786 km | 单向约 120 ms | 3 颗覆盖全球，时延大 |
| MEO 中轨 | ~2000~20000 km | 数十 ms | 导航（GPS/北斗） |
| LEO 低轨 | ~500~2000 km | 几 ms | 时延低，需大量卫星组网 |

### 介质综合对比

| 介质 | 带宽 | 传输距离 | 成本 | 抗干扰 |
|------|------|---------|------|--------|
| UTP 双绞线 | 中 | 短（~100 m/段） | 低 | 较弱 |
| 同轴电缆 | 中高 | 中 | 中 | 中 |
| 多模光纤 | 高 | 中（数百 m） | 中 | 强（免电磁干扰） |
| 单模光纤 | 极高 | 长（数十 km+） | 较高 | 强 |
| 无线 | 受频段限 | 视频段/功率 | 低（免布线） | 弱（易受干扰、被窃听） |

## 信号的编码与调制

数据有数字/模拟两类，信号也有数字/模拟两类，组合出四种转换。核心问题：怎么把待传数据"画"成信号波形。

```
            目标信号
            数字         模拟
        ┌──────────┬──────────┐
 数字   │  编码     │  调制     │
 数据   │ NRZ/曼彻  │ ASK/FSK   │
        │ 斯特/4B5B │ PSK/QAM   │
        ├──────────┼──────────┤
 模拟   │  PCM      │  调制     │
 数据   │ 采样量化  │ AM/FM/PM  │
        └──────────┴──────────┘
```

### 数字数据 → 数字信号（编码）

用不同电平/跳变表示 0 和 1。评判一个编码好坏看两点：**自同步能力**（接收端能否从信号本身恢复时钟）和**有无直流分量**（长串相同位会累积直流，对耦合电路不利）。

以比特流 `0 1 0 0 1 1` 为例（高=正电平，低=负电平/零）：

```
        bit:   0   1   0   0   1   1
              ─────────────────────────
 NRZ           ┌───┐       ┌───────┐      高=1 低=0
              ─┘   └───────┘       └──    无跳变靠电平区分

 RZ            ┌─┐         ┌─┐ ┌─┐        每位中途归零
              ─┘ └─────────┘ └─┘ └────

 曼彻斯特       ┌┐  ┌┐  ┌┐  ┌┐  ┌┐  ┌┐    位中点必跳变:
              ─┘└──┘└──┘└──┘└──┘└──┘└─    ↓跳=0 ↑跳=1(IEEE)

 差分曼彻斯特   位起点有跳变=0, 无跳变=1;  位中点恒有跳变(供同步)
```

- **NRZ（不归零）**：电平高表示 1、低表示 0，整位保持。最简单，但长串 0 或长串 1 时电平不变，接收端**无法从信号提取时钟（无自同步）**，且有直流分量。
- **RZ（归零）**：每位中途回到零电平。引入了跳变利于同步，但每位要两次跳变，**占用带宽翻倍**。
- **曼彻斯特编码**：每个比特中点必有一次跳变，"↑跳=1、↓跳=0"（IEEE 802.3 约定）。跳变既载数据又供时钟——**自同步、无直流分量**。代价是信号变化率是数据率 2 倍（波特率=2×比特率），带宽利用率低。**经典以太网（10BASE-T）用它**。
- **差分曼彻斯特**：靠"每位起点有无跳变"区分 0/1（有跳变=0，无=1），位中点恒有跳变供同步。优点是抗极性反转（线接反也能解），用于令牌环。

> **4B/5B 编码**：把每 4 个数据比特映射成 5 比特码字，码字精心挑选，保证不出现长串 0，从而具备自同步能力；再配合 NRZI 传输。代价是 25% 的额外开销（4 bit 传 5 bit）。用于 100BASE-FX/TX。
>
> 同理还有 **8B/10B**（千兆以太网/光纤通道），思路一致：用查表式块编码换取直流平衡与自同步。

易混：NRZ"省带宽但不自同步"，曼彻斯特"自同步但费带宽"，4B/5B 是两者的折中——只花 25% 开销就拿到自同步。

### 数字数据 → 模拟信号（调制）

用数字数据去改变载波 $A\cos(2\pi f t + \varphi)$ 的某个参数。基本三种 + 一种组合：

```
 数据:    1     0     1     1
       
 ASK   ∿∿∿  ────  ∿∿∿  ∿∿∿   幅度: 有载波=1, 无=0 (通断键控)

 FSK   ∿∿∿  ∼∼∼  ∿∿∿  ∿∿∿   频率: 高频=1, 低频=0

 PSK   ∿∿∿  ∿͞∿͞∿  ∿∿∿  ∿∿∿   相位: 相位翻转表示0/1
```

- **ASK（幅移键控）**：用幅度（有/无、强/弱）表示数据。实现简单但抗噪差。
- **FSK（频移键控）**：用不同频率表示数据。抗噪较好，早期电话拨号 Modem 用。
- **PSK（相移键控）**：用载波相位表示数据。抗噪最好。**DPSK（差分 PSK）** 用"相位是否变化"而非绝对相位表示数据，免去接收端的绝对相位基准。
- **QAM（正交幅度调制）**：同时调制**幅度和相位**，一个码元落在"星座图"上的某个点，点数越多每码元载比特越多。是现代高速通信（Wi-Fi、LTE、Cable）的主力。

星座图（以 16-QAM 为例，16 个点 = 每码元 4 bit）：

```
         Q
      ·  ·  ·  ·
      ·  ·  ·  ·
 ─────────────────► I
      ·  ·  ·  ·
      ·  ·  ·  ·
  每点=(幅度,相位)的一种组合, 共16点
  星座点越密 → 每码元比特越多, 但越易被噪声"认错"
```

注：QPSK 是 $M=4$ 的 PSK（4 个相位，每码元 2 bit）；16-QAM 每码元 4 bit、64-QAM 每码元 6 bit——这正是 $R_b=R_B\log_2 M$ 中靠增大 $M$ 提速的实现。

### 模拟数据 → 数字信号（PCM）

把模拟信号（如语音）数字化，三步走：**采样 → 量化 → 编码**。

```
 模拟波形 ──采样──> 离散样值 ──量化──> 取最近的量化级 ──编码──> 二进制码字
   ∿∿∿        | | | |          阶梯化               0110 1011 ...
```

1. **采样**：按固定间隔取模拟信号的瞬时值。采样频率必须满足**奈奎斯特采样定理**：

$$f_s \ge 2 f_{max}$$

其中 $f_{max}$ 是信号最高频率。低于此值会发生混叠（aliasing），无法还原原信号。

2. **量化**：把连续的样值取整到有限个量化级。$n$ 位量化 → $2^n$ 个级，位数越多越精细、量化噪声越小。
3. **编码**：把量化级用二进制码字表示。

PCM 输出的数据率：

$$R = f_s \times n \quad(\text{bps})$$

**举例（标准话音）**：电话话音带宽取 0~4 kHz，故 $f_s = 8000$ Hz；每样值量化 8 bit：

$$R = 8000 \times 8 = 64000 \text{ bps} = 64 \text{ kbps}$$

这正是数字电话一路 PCM 话音的 64 kbps，也是 T1/E1 时隙速率的由来。

### 模拟数据 → 模拟信号（调制）

把模拟基带信号搬移到高频载波上（为了便于天线辐射、频分复用）。同样改变载波的幅度/频率/相位：

- **AM（调幅）**：载波幅度随基带信号变化。带宽窄、抗噪差（调幅广播）。
- **FM（调频）**：载波频率随基带信号变化。抗噪好、占带宽（调频广播、音质更好）。
- **PM（调相）**：载波相位随基带信号变化，与 FM 密切相关。

### 编码/调制方式小结

| 转换 | 方式 | 关键点 |
|------|------|--------|
| 数字→数字 | NRZ / RZ / 曼彻斯特 / 4B5B | 看自同步能力与带宽开销 |
| 数字→模拟 | ASK / FSK / PSK / QAM | 增大 $M$ 提速，受噪声限制 |
| 模拟→数字 | PCM（采样-量化-编码） | 采样定理 $f_s\ge 2f_{max}$ |
| 模拟→模拟 | AM / FM / PM | 搬移到载波便于传输/复用 |

## 信道容量与传输极限

信道容量 = 信道能可靠传输的最大数据率。它由两条定理从两个角度给出上限。

### 奈奎斯特定理（无噪声、带宽受限）

理想无噪声信道，容量只受带宽和码元电平数限制：

$$C = 2W \log_2 M \quad(\text{bps})$$

- $W$：信道带宽（Hz）。
- $M$：每个码元的离散电平数。
- 物理含义：带宽 $W$ 的信道每秒最多承载 $2W$ 个码元（奈奎斯特码元速率上限），每码元载 $\log_2 M$ 比特。

注：无噪声时只要 $M$ 够大，容量似乎能无限提高——但真实信道总有噪声，$M$ 受噪声制约，这就要用香农定理。

### 香农定理（有噪声、信噪比受限）

有噪声信道，容量受带宽和信噪比共同限制：

$$C = W \log_2\left(1 + \frac{S}{N}\right) \quad(\text{bps})$$

- $S/N$：信噪比（信号功率 / 噪声功率，**线性值**）。
- 这是理论极限：无论用多复杂的编码/调制，实际速率都不可能超过 $C$。

信噪比常用分贝表示，注意计算时要先换回线性值：

$$\text{SNR(dB)} = 10\log_{10}\frac{S}{N} \quad\Longrightarrow\quad \frac{S}{N} = 10^{\frac{\text{SNR(dB)}}{10}}$$

dB 速查：10 dB→10 倍，20 dB→100 倍，30 dB→1000 倍，40 dB→10000 倍。

### 两定理综合使用

奈奎斯特给出"码元层面"的上限（取决于你打算用几电平 $M$），香农给出"信息层面"的物理极限（取决于噪声）。**实际可达速率取两者中的较小值**：

- 若按某个 $M$ 算出的奈奎斯特速率 **> 香农容量**：说明噪声不允许这么多电平被可靠区分，速率被香农限住，应降低 $M$。
- 提高容量的途径：加大带宽 $W$、提高信噪比 $S/N$、增大 $M$（但受香农制约）。

具体数值见[例题1](#例题1奈奎斯特香农联合容量)。

## 多路复用技术

多路复用（Multiplexing）：让多路信号共享一条物理信道，提高利用率。核心是给每路分配一份"互不干扰"的资源——频率、时间或码字。

```
 多路输入 ──> [ 复用器 MUX ] ══共享信道══> [ 分用器 DEMUX ] ──> 多路输出
   A,B,C,D                                                    A,B,C,D
```

### 频分复用 FDM / 波分复用 WDM

> **FDM（频分复用）**：把信道总带宽划成若干**频段**，每路信号占一个频段，各频段间留保护带防串扰。各路在频率上分开、时间上并行。

适合模拟信号，典型应用：有线电视、调频广播、DSL（话音与数据分频段）。

```
 频率
  │ ┌──┐保护带┌──┐保护带┌──┐
  │ │路1│      │路2│      │路3│
  └─┴──┴──────┴──┴──────┴──┴──> 各路同时传, 占不同频段
```

> **WDM（波分复用）**：FDM 在**光纤**上的实现——不同波长（颜色）的光承载不同信号，在一根光纤里并行传输。**DWDM**（密集波分）波长间隔小、容量极大；**CWDM**（粗波分）间隔大、成本低。

### 时分复用 TDM

> **TDM（时分复用）**：把时间切成周期性的帧，每帧再分若干**时隙**，每路信号占用固定时隙轮流发送。各路在时间上错开、共用全部带宽。

```
 时间 ──>
 帧1[ A | B | C | D ] 帧2[ A | B | C | D ] ...
      └时隙┘            每路每帧固定占一个时隙
```

- **同步 TDM（STDM）**：时隙**固定分配**给各路，即使某路没数据，它的时隙也空着浪费。简单、时延确定（电话网 T1/E1 用）。
- **统计 TDM（异步 TDM）**：时隙**按需动态分配**，谁有数据谁用，每个数据单元带上标识表明归属。利用率高（利用各路"不会同时满负荷"的特点），但需要寻址、可能排队。分组交换网本质就是统计复用。

易混：同步 TDM "对号入座"——空座也占；统计 TDM "先到先得"——按实际需求分。后者复用增益高，但放弃了固定时延保证。详见[例题5](#例题5tdm-与统计复用)。

### 码分复用 CDM / CDMA

> **CDMA（码分多址）**：各路同时占用全部频段和时间，靠互相**正交的扩频码（码片序列）** 区分。每个用户分到一个码，发送时把数据位与自己的码相乘扩频；接收时用同一个码做相关运算（内积）提取本路数据，其他路因码正交被抵消。

用 **Walsh 码**（来自哈达玛矩阵）作正交码。约定：发比特 1 → 发送 `+码`，发比特 0 → 发送 `-码`，不发 → 0。各码满足：**自身内积 = 码长，互相内积 = 0**。

恢复时：用某用户的码与信道叠加信号做内积，再除以码长，结果 $+1$ 判为 1、$-1$ 判为 0、$0$ 判为未发送。完整计算见[例题4](#例题4cdma-编解码)。

```
 站A数据 ─×码A─┐
 站B数据 ─×码B─┼─叠加─> 共享信道 ─┬─×码A÷码长─> 还原A
 站C数据 ─×码C─┘                   ├─×码B÷码长─> 还原B
                                    └─×码C÷码长─> 还原C
   (码相互正交, 内积时彼此抵消)
```

注：扩频还带来抗干扰、抗截获、低功率谱密度的好处，曾用于 2G/3G 蜂窝和军用通信。

### 正交频分复用 OFDM（简介）

OFDM 把信道分成大量**相互正交的窄带子载波**并行传输。子载波频谱虽彼此重叠却因正交而不干扰，频谱利用率远高于传统 FDM；每个子载波速率低、抗多径能力强。是 Wi-Fi（802.11a/g/n/ac）、LTE、5G、数字电视的核心物理层技术。

## 物理层设备与接口

### 物理层设备

物理层设备只处理比特/信号本身（放大、整形、转发），不看也不懂帧或地址。

- **中继器（Repeater）**：接收衰减/失真的信号，**整形放大后再生转发**，用于延长传输距离。只有两个端口。
- **集线器（Hub）**：本质是**多端口中继器**。从一个端口收到的比特，向所有其他端口广播。

> **与冲突域的关系**：集线器只是把信号广播出去，所有接它的主机仍处于**同一个冲突域**——任意两台同时发送就会冲突。它不隔离冲突、不缓存、不看地址，这是它与"交换机"的本质区别。交换机工作在链路层、每端口独立冲突域，详见 [6.4 交换局域网](6.4链路层：交换局域网.md)。

```
 集线器(物理层)：一冲突域           交换机(链路层)：每端口独立冲突域
   A ─┐                              A ─┤端口
   B ─┼─[Hub]── 同一冲突域           B ─┤端口  各端口隔离, 可全双工
   C ─┘  (广播,会冲突)               C ─┤端口
```

### 物理层接口的四大特性

物理层接口（如网卡与线缆的连接点）的规范从四个维度定义：

| 特性 | 规定什么 | 例子 |
|------|---------|------|
| 机械特性 | 接口形状、尺寸、引脚数与排列 | RJ-45 水晶头 8 针、DB-9 接口 |
| 电气特性 | 电压范围、电平、阻抗、信号速率 | RS-232 用 ±3~±15 V 表示 0/1 |
| 功能特性 | 各引脚（线）的功能含义 | 哪根线是发送、接收、地、控制 |
| 规程特性 | 信号时序、动作的先后顺序 | 建立/维持/释放连接的事件序列 |

注：四特性可记成"形（机械）、压（电气）、用（功能）、序（规程）"——一个引脚长什么样、上面多少伏、这伏特代表什么、什么时候发。

### 常见物理层标准

- **EIA RS-232**：经典串行通信接口标准（机械 DB-9/DB-25、电气 ±电平、功能引脚定义、规程握手）。
- **RJ-45**：双绞线以太网常用的 8 针接口（机械特性）。
- **以太网物理标准**（命名 `速率 BASE 介质`）：

| 标准 | 速率 | 介质 | 编码 |
|------|------|------|------|
| 10BASE-T | 10 Mbps | Cat3+ 双绞线 | 曼彻斯特 |
| 100BASE-TX | 100 Mbps | Cat5 双绞线 | 4B/5B + NRZI |
| 1000BASE-T | 1 Gbps | Cat5e 双绞线 | 8B/1Q（PAM-5，四对线并行） |
| 1000BASE-LX | 1 Gbps | 单模/多模光纤 | 8B/10B |

> 命名中 `BASE` 表示基带传输，`T` 为双绞线、`X/LX/SX` 为光纤等。

## 典型例题

> 以下数值均用 Python 验算。

### 例题1：奈奎斯特/香农联合容量

**题**：某信道带宽 $W = 3000$ Hz，信噪比为 30 dB。(1) 求该信道的理论最大数据率；(2) 若要用奈奎斯特公式达到该速率，每码元应取多少电平 $M$？

**解**：

(1) 先把 dB 换成线性信噪比：

$$\frac{S}{N} = 10^{\frac{30}{10}} = 10^3 = 1000$$

香农容量：

$$C = W\log_2\left(1+\frac{S}{N}\right) = 3000 \times \log_2(1+1000) = 3000 \times 9.967 \approx 29902 \text{ bps} \approx 29.9 \text{ kbps}$$

(2) 香农允许的每码元比特数：

$$\log_2 M = \frac{C}{2W} = \frac{29902}{2\times 3000} \approx 4.98 \text{ bit/码元}$$

故 $M \le 2^{4.98} \approx 31.6$，实际取 $M = 16$（4 bit）安全；若取 $M=32$（5 bit），奈奎斯特速率 $2\times3000\times5 = 30000$ bps 已略超香农极限，无法可靠传输。**实际速率取两者较小值**，受香农限约 29.9 kbps。

**Python 验算**：

```python
import math
W = 3000
snr = 10**(30/10)            # 1000.0
C = W*math.log2(1+snr)       # 29901.68 bps
bits_per_symbol = C/(2*W)    # 4.984 bit
# 输出: C≈29901.7 bps≈29.9kbps; 每码元≤4.98bit, 取 M=16 安全
```

验算结果：$S/N=1000$，$C \approx 29901.7$ bps $\approx 29.9$ kbps，每码元 $\le 4.98$ bit。

### 例题2：PCM 数据率

**题**：(1) 对带宽 0~4 kHz 的话音按奈奎斯特采样、8 bit 量化做 PCM，求数据率；(2) 对 CD 音质（采样率 44.1 kHz、16 bit、双声道立体声）求数据率。

**解**：

(1) 采样定理：$f_s \ge 2f_{max} = 2\times 4000 = 8000$ Hz，取 $f_s = 8000$ Hz。

$$R = f_s \times n = 8000 \times 8 = 64000 \text{ bps} = 64 \text{ kbps}$$

(2) 双声道需乘 2：

$$R = 44100 \times 16 \times 2 = 1411200 \text{ bps} \approx 1.41 \text{ Mbps}$$

**Python 验算**：

```python
print(8000*8)          # 64000  -> 64 kbps (标准PCM话音)
print(44100*16*2)      # 1411200 -> ≈1.41 Mbps (CD音质)
```

验算结果：话音 PCM = 64 kbps（即数字电话一路速率）；CD = 1411.2 kbps ≈ 1.41 Mbps。

### 例题3：电话信道香农容量

**题**：一条电话线带宽 $W = 3.1$ kHz，信噪比 20 dB，求理论最大数据率。

**解**：

$$\frac{S}{N} = 10^{20/10} = 100$$
$$C = 3100 \times \log_2(1+100) = 3100 \times 6.658 \approx 20640 \text{ bps} \approx 20.6 \text{ kbps}$$

**Python 验算**：

```python
import math
print(3100*math.log2(1+10**(20/10)))   # 20640.5 bps ≈ 20.6 kbps
```

验算结果：$C \approx 20640.5$ bps $\approx 20.6$ kbps。这解释了为何老式电话拨号 Modem 的极限约在 30 kbps 上下——受电话信道带宽与信噪比的物理极限制约。

### 例题4：CDMA 编解码

**题**：4 个站使用 4 阶 Walsh 正交码：

$$A=(+1,+1,+1,+1),\ B=(+1,-1,+1,-1),\ C=(+1,+1,-1,-1),\ D=(+1,-1,-1,+1)$$

某时刻 A 发送比特 1、B 发送比特 0、C 未发送、D 发送比特 1。求信道叠加信号，并验证各站能正确恢复自己的数据。

**解**：

约定发 1 用 `+码`、发 0 用 `-码`、不发为 0。各站发送的码片：

$$+A=(+1,+1,+1,+1),\quad -B=(-1,+1,-1,+1),\quad +D=(+1,-1,-1,+1)$$

信道叠加（逐位相加）：

$$\text{信道} = (+A)+(-B)+(+D) = (1,\ 1,\ -1,\ 3)$$

接收方用各码与信道做内积再除以码长 4：

- 恢复 A：$(1,1,-1,3)\cdot A = 1+1-1+3 = 4$，$4/4 = +1 \Rightarrow$ **比特 1**
- 恢复 B：$(1,1,-1,3)\cdot B = 1-1-1-3 = -4$，$-4/4 = -1 \Rightarrow$ **比特 0**
- 恢复 C：$(1,1,-1,3)\cdot C = 1+1+1-3 = 0$，$0 \Rightarrow$ **未发送**
- 恢复 D：$(1,1,-1,3)\cdot D = 1-1+1+3 = 4$，$4/4 = +1 \Rightarrow$ **比特 1**

四站全部正确恢复，互不干扰——这正是正交码的威力。

**Python 验算**：

```python
A=[1,1,1,1]; B=[1,-1,1,-1]; C=[1,1,-1,-1]; D=[1,-1,-1,1]
inner=lambda x,y: sum(a*b for a,b in zip(x,y))
chan=[a-b+d for a,b,d in zip(A,B,D)]          # +A -B +D = [1,1,-1,3]
for n,c in [('A',A),('B',B),('C',C),('D',D)]:
    print(n, inner(chan,c)//4)                # A:1  B:-1  C:0  D:1
```

验算结果：信道 = `[1,1,-1,3]`；恢复 A=+1、B=−1、C=0、D=+1，与发送完全一致。

### 例题5：TDM 与统计复用

**题**：(1) 用同步 TDM 复用 24 路 64 kbps 话音，数据部分总速率是多少？这对应哪条标准线路？(2) 设有 100 个用户、每户峰值 100 kbps，但平均只有 10% 时间在传（占空比 0.1）。对比同步 TDM 与统计 TDM 各需多少带宽。

**解**：

(1) 同步 TDM 数据速率 = 路数 × 每路速率：

$$24 \times 64 = 1536 \text{ kbps} = 1.536 \text{ Mbps}$$

T1 线路还要给每帧加 1 个帧同步位：每帧 $24\times 8 + 1 = 193$ bit，8000 帧/秒，故 $193 \times 8000 = 1.544$ Mbps——这正是 **T1** 的 1.544 Mbps。

(2) 同步 TDM 必须按**峰值**为每路预留固定时隙：

$$100 \times 100 \text{ kbps} = 10000 \text{ kbps} = 10 \text{ Mbps}$$

统计 TDM 按**实际平均需求**分配：

$$100 \times 100 \times 0.1 = 1000 \text{ kbps} = 1 \text{ Mbps}$$

理想复用增益 $\approx 1/0.1 = 10$ 倍。代价是突发并发时可能排队/丢包，失去固定时延保证。

**Python 验算**：

```python
print(24*64)              # 1536 kbps (数据)
print((24*8+1)*8000)      # 1544000 -> 1.544 Mbps (T1含帧位)
print(100*100)            # 10000 kbps = 10 Mbps (同步,按峰值)
print(int(100*100*0.1))   # 1000 kbps = 1 Mbps (统计,按平均)
```

验算结果：同步 TDM 数据 1.536 Mbps、T1 线路 1.544 Mbps；100 用户场景下同步需 10 Mbps、统计仅需约 1 Mbps，复用增益约 10 倍。
