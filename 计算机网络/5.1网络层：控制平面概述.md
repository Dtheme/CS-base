## 控制平面的基本功能

### 控制平面的定义与作用

> **网络层控制平面 (Control Plane)**
> 
> 决定分组从源到目的端沿途经过哪条路径，即计算并维护每台路由器的转发表/路由表。与之相对，数据平面负责按表逐包转发。

控制平面的职责：

- 维护网络拓扑信息
- 计算到各目的网络的最优路径
- 生成并更新路由表
- 在链路或节点变化时重新计算

```
主机A
   │  控制平面预先算好路由表
   ▼
R1 ──→ R2 ──→ R3      数据平面按表逐跳转发
   │
   ▼
主机B
```

注：控制平面"算路"，数据平面"转发"。路由表是控制平面的输出、数据平面的输入。

### 控制平面与数据平面的协同

| 维度 | 控制平面 | 数据平面 |
|------|----------|----------|
| 主要任务 | 计算路由路径 | 转发数据包 |
| 时间尺度 | 秒到分钟级 | 纳秒到微秒级 |
| 触发频率 | 拓扑变化时 | 每个分组 |
| 复杂度 | 算法复杂 | 查表简单 |
| 实现方式 | 软件 | 硬件加速 |
| 典型代表 | OSPF、BGP | 最长前缀匹配转发 |

两个平面的数据流向：控制平面产出路由表，下沉为数据平面的转发表；数据平面据此处理每个到达的分组。

```
控制平面：拓扑发现 → 路由计算 → 生成路由表
                                  │ 下沉
                                  ▼
数据平面：转发表 → 逐包查表 → 转发到输出端口
```

### 控制平面的两种组织范式

教材给出两类控制平面实现方式：每路由器控制（传统分布式）与逻辑集中控制（SDN）。

**每路由器控制（传统分布式）**：控制平面与数据平面同处一台路由器，每台路由器运行路由算法，彼此通过路由协议交换信息，分布式地算出路由表。

```
   ┌─────────────┐      ┌─────────────┐      ┌─────────────┐
   │   路由器A    │      │   路由器B    │      │   路由器C    │
   │ ┌─────────┐ │      │ ┌─────────┐ │      │ ┌─────────┐ │
   │ │ 控制平面 │◄┼──────┼►│ 控制平面 │◄┼──────┼►│ 控制平面 │ │
   │ └────┬────┘ │ 路由  │ └────┬────┘ │ 路由  │ └────┬────┘ │
   │ ┌────┴────┐ │ 协议  │ ┌────┴────┐ │ 协议  │ ┌────┴────┐ │
   │ │ 数据平面 │ │      │ │ 数据平面 │ │      │ │ 数据平面 │ │
   │ └─────────┘ │      │ └─────────┘ │      │ └─────────┘ │
   └─────────────┘      └─────────────┘      └─────────────┘
   每台路由器自带控制平面，分布式决策、分布式执行
```

**逻辑集中控制（SDN）**：控制平面从路由器中剥离，集中到远程控制器；交换机只保留数据平面，按控制器经控制协议（如 OpenFlow）下发的流表转发。

```
            ┌──────────────────┐
            │   远程SDN控制器   │   逻辑集中的控制平面
            └───┬─────┬─────┬──┘
       控制协议 │     │     │ （如 OpenFlow）
            ┌───▼─┐ ┌─▼───┐ ┌▼────┐
            │交换机│ │交换机│ │交换机│   仅数据平面
            │ A   │ │ B   │ │ C   │   按下发流表转发
            └─────┘ └─────┘ └─────┘
```

易混："逻辑集中"指控制逻辑统一，物理上控制器仍可由多台服务器组成集群以保证可靠性与扩展性，并非真的只有一台机器。

## 路由选择的基本概念

### 路由与转发的区别

| 概念 | 英文 | 含义 | 作用范围 | 时间尺度 |
|------|------|------|----------|----------|
| 路由选择 | Routing | 确定分组从源到目的的整条路径 | 端到端 | 相对静态 |
| 转发 | Forwarding | 把分组从输入端口送到合适的输出端口 | 单台路由器 | 实时逐包 |

注：路由选择是控制平面的工作（算出路径），转发是数据平面的工作（执行查表）。可类比导航——路由是规划整条路线，转发是司机在每个路口按指示牌选出口。

### 路由选择问题的图抽象

路由选择问题可抽象为带权图 $G = (N, E)$：

- $N$：节点集合，每个节点对应一台路由器
- $E$：边集合，每条边对应一条直连链路
- $c(x, y)$：边 $(x, y)$ 的代价（链路代价），不直连时取 $\infty$

```
        2          1
   u ──────── v ──────── z
   │          │          │
  5│         3│         2│
   │          │          │
   x ──────── w ──────── y
        1          3

节点 = 路由器     边 = 链路     边上数字 = 链路代价

如 u→x→w→v 的代价 = 5 + 1 + 3 = 9
```

求最优路由即在该图上求从源节点到目的节点的最小代价路径。一条路径的代价为其各边代价之和：

$$\text{cost}(\text{path}) = c(x_1, x_2) + c(x_2, x_3) + \cdots + c(x_{p-1}, x_p)$$

注：代价对称（$c(x,y)=c(y,x)$）只是常见简化，并非必然；实际链路两个方向的代价可以不同。

### 链路代价的定义

链路代价由网络管理员设定，可反映不同的优化目标：

| 代价依据 | 含义 | 倾向选择 |
|----------|------|----------|
| 跳数 | 每条链路代价记为 1 | 经过路由器最少的路径 |
| 带宽 | 与链路带宽成反比 | 高带宽链路（OSPF 默认做法） |
| 时延 | 传播 + 排队时延 | 低时延路径 |
| 负载 | 随链路利用率变化 | 避开拥塞链路 |

注：代价取常数（如跳数、按带宽算的固定值）时与流量无关，称为负载不敏感；代价随实时利用率变化则为负载敏感，后者可能引起路由振荡，故实际协议多采用前者。

## 路由算法的分类

### 按信息完整性分类

这是教材的主分类维度，依据每个节点掌握的网络信息多少分为两类。

#### 全局路由算法（链路状态算法）

> **链路状态算法 (Link State, LS)**
> 
> 每个节点都掌握完整的网络拓扑与所有链路代价，基于全局信息独立计算最短路径。

- 信息获取：通过链路状态广播，每个节点把自己的链路代价洪泛到全网，于是各节点都得到一致的全局拓扑
- 计算方式：每个节点独立运行 Dijkstra 算法求最短路径树
- 代表协议：OSPF、IS-IS

#### 分布式路由算法（距离向量算法）

> **距离向量算法 (Distance Vector, DV)**
> 
> 每个节点只知道到各邻居的链路代价，通过与邻居交换"到各目的的最小代价"逐步迭代收敛。

- 信息获取：只从直连邻居处获得其距离向量，不掌握全局拓扑
- 计算方式：基于 Bellman-Ford 方程做分布式迭代
- 代表协议：RIP、（早期）BGP

Bellman-Ford 方程：

$$d_x(y) = \min_v \{\, c(x,v) + d_v(y) \,\}$$

其中 $d_x(y)$ 为 $x$ 到 $y$ 的最小代价，$c(x,v)$ 为 $x$ 到邻居 $v$ 的链路代价，$d_v(y)$ 为邻居 $v$ 到 $y$ 的最小代价。

注：LS 与 DV 的具体执行过程见 [5.2 路由选择算法](5.2网络层：路由选择算法.md)。

### 按是否随负载变化分类

| 类型 | 链路代价 | 特点 |
|------|----------|------|
| 负载敏感 | 随实时链路利用率变化 | 能避开拥塞，但易引起路由振荡 |
| 负载不敏感 | 与当前负载无关（如跳数、按带宽算的固定值） | 路由稳定，当今 Internet 协议（RIP、OSPF、BGP）均属此类 |

### 按是否自动适应拓扑分类

| 类型 | 路由表来源 | 适用场景 |
|------|-----------|----------|
| 静态路由 | 人工配置，很少变动 | 小型、拓扑稳定的网络；备份路径 |
| 动态路由 | 运行路由协议，随拓扑变化自动重算 | 大型、变化频繁的网络 |

## 传统路由与SDN路由

两种控制平面范式的取舍，本质是"分布式自治"与"集中可编程"之间的权衡。

| 维度 | 每路由器控制（传统） | 逻辑集中控制（SDN） |
|------|---------------------|---------------------|
| 控制平面位置 | 分散在各路由器内 | 集中于远程控制器 |
| 鲁棒性 | 高，无中心，单点故障影响局部 | 控制器是单点，需主备/集群冗余 |
| 扩展性 | 各节点独立处理，无中心瓶颈 | 受控制器性能限制 |
| 全局优化 | 难，缺乏全局视图 | 易，基于完整拓扑统一决策 |
| 策略管理 | 需逐台配置，策略分散 | 集中制定与下发 |
| 演进速度 | 受协议标准化制约，较慢 | 软件实现，便于快速创新 |
| 成熟度 | OSPF、BGP 等长期部署 | 数据中心广泛采用，仍在演进 |

注：SDN 把控制逻辑移到通用服务器上的软件，从而摆脱了路由器厂商私有实现的束缚，这是它便于创新、便于做细粒度流量工程的根本原因。

实际网络常按场景混合使用：核心网与广域互联仍以传统分布式协议为主（看重稳定与成熟），数据中心内部多用 SDN（看重灵活与集中优化），两者通过控制器与传统协议协作实现渐进演进。
